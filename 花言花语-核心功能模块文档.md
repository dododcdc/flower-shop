# 🌺 【花言花语】鲜花售卖系统 - 核心功能模块

## 项目概述
为【花言花语】花店开发的在线销售平台，扩展10km范围内的线上销售渠道，实现从纯线下门店到线上线下结合的经营模式升级。

## 现状分析
- **当前业务**：纯线下门店销售，客户到店购买
- **经营痛点**：只覆盖到店客户，人工统计库存
- **经营花材**：玫瑰、百合、夜来香、康乃馨、向日葵
- **包装形式**：花束、花篮
- **服务范围**：门店10km范围内配送
- **主要场景**：生日、情人节、纪念日、看望病人

## 实际业务流程

**线下现有流程**：
客户到店 → 选择花束/花篮 → 当面付款 → 当场交易

**线上新增流程**：
老板登录 → 上架商品 → 线上客户浏览 → 游客下单 → 老板配送 → 客户确认收货

## 核心功能模块

### 1. 后台管理系统（老板端）
**功能描述**：花店老板的线上店铺管理中心

**主要功能**：
- **管理员登录**：首次访问引导设置管理员账号
- **商品管理**：上架/下架花束和花篮，设置价格、描述、图片
- **库存管理**：设置商品库存数量，模拟"挂牌子"方式
- **订单处理**：查看新线上订单，获取客户信息
- **配送管理**：确认配送范围，查看配送地址
- **店铺设置**：【花言花语】店名、联系电话、门店地址

**设计重点**：
- 界面简单，像你现在的记账方式一样直观
- 快速上架新到的花束
- 及时收到线上订单通知（声音提醒）

### 2. 线上商品展示模块
**功能描述**：让10km范围内的客户能看到你的花

**商品分类**：
- 按花材分类：玫瑰、百合、夜来香、康乃馨、向日葵
- 按包装分类：花束、花篮
- 按场景分类：生日、情人节、纪念日、看望病人

**展示内容**：
- 清晰的花束/花篮图片
- 价格信息（花篮比花束贵）
- 库存状态（有货/售罄）
- 简单的花语和寓意介绍

**展示特点**：
- 突出【花言花语】的品牌特色
- 图片真实，和实际商品一致
- 价格透明，包含配送费说明

### 3. 游客下单模块
**功能描述**：让客户无需注册就能方便地在线购买

**下单流程**：
1. 选择花束/花篮（玫瑰花束、百合花篮等）
2. 填写配送地址（系统判断是否在10km范围内）
3. 填写收货人信息（姓名、联系电话）
4. 选择配送时间（今天/明天，具体时段）
5. 确认订单和总价
6. 在线支付（支付宝/微信支付）

**关键功能**：
- **无需注册**：降低购买门槛
- 地址验证（超出10km范围提醒无法配送）
- 时间选择（给你足够的准备时间）
- 费用透明（商品价格+配送费）

### 4. 购物车模块
**功能描述**：让游客客户能一次购买多个商品

**主要功能**：
- **添加商品**：选择多个花束/花篮加入购物车
- **管理购物车**：修改数量、删除商品、清空购物车
- **价格计算**：自动计算商品总价和配送费
- **库存检查**：实时显示商品库存状态
- **批量结算**：一次性结算购物车中的所有商品

**使用场景**：
- 客户想买"玫瑰花束+百合花篮"组合
- 节日期间需要多个花束
- 为多个场合一次性购买

### 5. 订单跟踪模块
**功能描述**：线上订单的全流程管理

**客户功能**：
- 通过订单号和手机号查询订单状态
- 查看订单状态：已付款→准备中→配送中→已完成
- 联系花店（电话/微信）

**老板功能**：
- 新订单提醒（声音+弹窗提醒）
- 查看订单详情（商品、地址、时间、客户电话）
- 更新订单状态（确认接单→准备花束→开始配送→完成）
- 获取客户联系方式（配送前电话确认）

**状态管理**：
- 已付款：客户下单完成，等待你确认
- 准备中：你开始准备花束
- 配送中：你开始配送
- 已完成：客户收到花

### 6. 用户系统（第三期功能）
**功能描述**：为老客户提供更好的购物体验

**注册/登录**：
- 可选注册/登录（游客也能购买）
- 手机号验证注册
- 简单的个人信息设置

**会员功能**：
- 保存收货地址
- 查看历史订单
- 收藏喜欢的商品
- 生日提醒服务

### 7. 库存管理模块（集成在商品管理中）
**功能描述**：模拟你现在的挂牌子方式，自动化管理

**库存管理对象**：
- 玫瑰花束、玫瑰花篮
- 百合花束、百合花篮
- 夜来香花束、夜来香花篮
- 康乃馨花束、康乃馨花篮
- 向日葵花束、向日葵花篮

**核心功能**：
- **设置库存**：每天进货后在系统里更新库存数量
- **自动扣减**：线上订单付款成功后自动减去对应数量
- **售罄提示**：库存为0时商品显示"暂时售罄"
- **低库存提醒**：库存少于3个时提醒补货

**管理方式**：
- 就像你现在挂"已售完"牌子一样简单
- 每天早上更新库存（根据进货情况）
- 系统自动帮你"摘牌子"（有货时恢复上架）

## 模块交互关系

```
你的日常操作流程：
登录后台 → 更新库存 → 处理新订单 → 准备花束 → 配送完成

客户操作流程：
浏览商品 → 加入购物车 → 填写地址 → 在线支付 → 等待收货

库存联动：
进货后 → 系统更新库存
客户购买 → 自动扣减库存
库存为0 → 自动显示售罄
补货后 → 重新显示有货

购物车与订单：
多个商品 → 加入购物车 → 统一结算 → 生成一个订单

用户与订单：
游客下单 → 通过订单号查询
注册用户 → 登录查看所有订单
```

## 关键业务规则

### 商品管理规则
- 每种花材分花束和花篮两个品类
- 花篮价格高于花束
- 库存按具体商品管理（如"11朵玫瑰花束"）
- 支持临时下架（暂时没货时）

### 游客下单规则
- 无需注册即可购买
- 必须填写姓名、电话、地址
- 通过订单号+手机号查询订单状态
- 支持后期注册关联历史订单

### 购物车规则
- 游客也能使用购物车
- 可以添加不同类型的商品
- 数量修改实时更新价格
- 库存不足时提醒客户

### 配送规则
- 仅限10km范围内配送
- 地址自动验证，超出范围提示无法配送
- 配送时间选择（给你准备时间）
- 配送前电话确认

### 订单规则
- 支付成功后才扣减库存
- 新订单立即通知你
- 客户通过订单号查询状态
- 支持按配送时间排序订单
